# InfraNexus Project Instructions

You are working on **InfraNexus**, a ServiceNow CMDB Graph Visualizer. This file applies to ALL files in the project.

## Quick Context

- **Backend**: FastAPI + Kuzu (embedded graph DB) + Redis + Meilisearch
- **Frontend**: Next.js 15 + React 19 + Three.js (react-force-graph-3d) + Zustand + shadcn/ui
- **Purpose**: Render 1M+ CMDB CIs as interactive 3D force-directed graph
- **Local-first**: Runs entirely via Docker Compose on a single machine

## Absolute Rules

1. **Kuzu is single-writer, single-process**. Never suggest multiple uvicorn workers.
2. **Parameterize all Cypher queries**. Never use f-strings or `.format()` in Kuzu queries.
3. **Dynamic import Three.js** with `ssr: false`. Never import Three.js at module level in Next.js.
4. **Max 2000 nodes** in any graph response. Always enforce `max_nodes` and `degree_threshold`.
5. **ServiceNow credentials go in `.env.local` only**. Never hardcode, never log, never commit.
6. **sys_id is always 32 hex chars**. Validate at API boundaries.
7. **Redis cache keys follow pattern**: `{type}:{id}:{params_hash}`. Always set TTL.
8. **All API responses have `Server-Timing` header** with query duration.
9. **Frontend calls backend directly** (CORS). No Next.js API proxy routes.
10. **Use Maps (not arrays) for node/edge storage** in frontend stores.

## File-Specific Guidance

- `backend/app/core/kuzu_manager.py` - Single writer with asyncio.Lock, pool of readers
- `backend/etl/snow_client.py` - Rate limit to 1.4 req/s, exponential backoff
- `backend/app/routers/graph.py` - Always return GraphResponse model with query_time_ms
- `frontend/src/components/graph/` - Three.js rendering, must handle dispose() for memory
- `frontend/src/lib/graphMerge.ts` - Preserve existing node positions on merge
- `frontend/src/store/graphStore.ts` - Nodes/edges as Map<string, T>
